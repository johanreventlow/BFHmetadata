### Casing
`id` vs `Id` varierer pr. tabel. Med quoted identifiers SKAL hver FK-reference
matche parent-PK'ens eksakte casing. FK-listen ovenfor bevarer eksakt casing.

### Cascade
Flere FK'er har "Overlappende opdateringer" (ON UPDATE CASCADE). PK'er er
autonumber/immutable → cascade moot. Konservativt: **udelad cascades** i v1.

## Script-design: `01_generate_ddl.R`

1. **Læs** `access_schema.yaml` (yaml-pakke) + `config.yml` (config-pakke)
2. **Hardkod FK-map + PK-map** som R-lister (kilde: PDF, manuelt verificeret ovenfor)
   — ikke parse PDF i scriptet (skrøbeligt); FK-listen er statisk og verificeret
3. **Type-map-funktion** `map_odbc_type(code)` med fail-loud på ukendt kode
4. **Per tabel, generér:**
   - `CREATE TABLE "tblXxx" (...)` med quoted identifiers (bevar æ/ø/å + casing)
   - PK-kolonne (`id`/`Id`, type 4) → `INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY`
     (tillader eksplicit Id-insert under data-import → sequence resettes i Trin 2)
   - Øvrige kolonner med mappet type
5. **Efter alle CREATE TABLE: ALTER TABLE ADD FOREIGN KEY** (separat blok, så
   load-rækkefølge er ligegyldig). De 16 gennemtvungne. Den ikke-gennemtvungne
   som **kommenteret** SQL + note (aktiveres i Trin 2 efter orphan-tjek).
6. **Skriv** `01_schema.sql` (UTF-8, LF) + log til konsol

## Åbne verifikations-punkter (afklares under script-skrivning, ej blokerende)

- **Junction-tabel-PK:** har tblForbind*-tabeller egen `Id`-PK eller komposit?
  Tjekkes i YAML/PDF Tabelindeks når scriptet skrives.
- **NOT NULL:** Documenter har "Required"-flag pr. felt — kan udtrækkes for
  NOT NULL-constraints. Beslutning: v1 konservativ (kun PK NOT NULL) eller
  fuld Required-mapping? Forslag: udtræk Required hvor trivielt, ellers defer.
